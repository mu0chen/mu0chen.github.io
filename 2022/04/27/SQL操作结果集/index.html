SQL 中的联表操作以及求并集、差集、交集

0. 需求背景

今天碰到了一个关于 SQL 中的问题，将其简易化之后即可得到这样一个关系：

即表 A-表 B，表 B-表 C，表 C-表 D，表 A-表 E，表 E-表 F 均为一对多关系，现要查询出表 A 中所有满足日期不早于给定日期的记录，并找出最晚的记录时间，即A 中记录对应的所有 D 和 F 的记录的日期均需要满足相应的日期条件。
以下是测试所用的数据：

insert into table_a VALUES(1, 'test_ok_name'),(2, 'test_error_name');

insert into table_b VALUES(1, 1), (2, 1), (3, 2), (4, 2);

insert into table_c VALUES(1, 'nameC1', 1), (2, 'nameC2', 1), (3, 'nameC3', 2), (4, 'nameC4', 2), (5, 'nameC5', 3), (6, 'nameC6', 3), (7, 'nameC7', 4), (8, 'nameC8', 4);

insert into table_d VALUES(1, 1, '2022-4-2'),(2, 1, '2022-4-3'),(3, 2, '2022-4-4'),(4, 2, '2022-4-5'),(5, 3, '2022-4-6'),(6, 3, '2022-4-7'),(7, 4, '2022-4-8'),(8, 4, '2022-4-9'),
                                                  (9, 5, '2022-4-3'),(10, 5, '2022-4-4'),(11, 6, '2022-4-5'),(12, 6, '2022-4-6'),(13, 7, '2022-4-7'),(14, 7, '2022-4-8'),(15, 8, '2022-4-9'),(16, 8, '2022-4-10');

insert into table_e VALUES(1, 1, 'nameE1'), (2, 1, 'nameE2'), (3, 2, 'nameE3'), (4, 2, 'nameE4');

insert into table_f VALUES(1, 1, '2022-4-2'),(2, 1, '2022-4-3'),(3, 2, '2022-4-4'),(4, 2, '2022-4-5'),
                                                  (5, 3, '2022-4-3'),(6, 3, '2022-4-4'),(7, 4, '2022-4-5'),(8, 4, '2022-4-6');

当我们需要求最晚记录时间为 2022-04-09 的 table_a 的记录，那么按照数据来说我们应该只求出 name 为 task_ok_name 的记录

1. 解决和分析

首先，这是一个典型的求数据库交集的问题，由于其中的关系均为一对多关系，因此，可以将表 A、B、C、D 联立起来，并把表 A、E、F 联立起来，求笛卡尔积，然后通过 HAVING 的方式进行筛选：

SELECT a.a_id from table_a a, table_b b, table_c c,table_d d
  where a.a_id = b.a_id and
        b.b_id = c.b_id and
        c.c_id = d.c_id
  GROUP BY a.a_id HAVING MAX(d.d_date) <= '2022-4-9';

SELECT a.a_id from table_a a, table_e e, table_f f
  where a.a_id = e.a_id and
        e.e_id = f.e_id
  GROUP BY a.a_id HAVING MAX(f.f_date) <= '2022-4-9';

再将两者求交集（注意：MySQL 中并没有 INTERSECT 关键字，因此不能用这个求交集，采用 inner join 的方式求交集）：

SELECT t1.a_id from
  (SELECT a.a_id from table_a a, table_b b, table_c c,table_d d
     where a.a_id = b.a_id and
           b.b_id = c.b_id and
           c.c_id = d.c_id
     GROUP BY a.a_id HAVING MAX(d.d_date) <= '2022-4-9') t1
  INNER JOIN
  (SELECT a.a_id from table_a a, table_e e, table_f f
     where a.a_id = e.a_id and
           e.e_id = f.e_id
     GROUP BY a.a_id HAVING MAX(f.f_date) <= '2022-4-9') t2
  on t1.a_id = t2.a_id;

2. 总结

2.1 SQL 求交集

SELECT column1 [, column2 ]
FROM table1 [, table2 ]
[WHERE condition]

INTERSECT

SELECT column1 [, column2 ]
FROM table1 [, table2 ]
[WHERE condition]

MySQL 不支持 INTERSECT 语句，因此一般采用 INNER JOIN 的方式来求交集：

SELECT DISTINCT
   id
FROM t1
   INNER JOIN t2 USING(id);

其中：

  • INNER JOIN 子句从左表和右表返回所有符合条件的行记录。
  • DISTINCT 运算符删除重复行。

2.2 SQL 求并集

-- UNION、UNION ALL能对两个或以上的结果集进行合并
-- 不包括重复行，进行默认排序
select a.id, a.name from tableA a
UNION
select b.id, b.name from tableB b

-- 包括重复行，不进行默认排序
select a.id, a.name from tableA a
UNION ALL
select b.id, b.name from tableB b

2.3 SQL 求差集

select a.id, a.name from tableA a
MINUS
select b.id, b.name from tableB b

同样，MySQL 并不支持差集 MINUS 关键字，因此，通常有两种做法：

  1. 采用 not in 的方式（效率太差，数据较大时容易出现死机的情况）
  2. 采用左连接 left join 的方式（如下，从 tableA 中剔除 tableB 中的数据）
select id from tableA a left join
(select id as tmp_id from tableB ) as b
on a.id = b.tmp_id where b.tmp_id is not NULL;
